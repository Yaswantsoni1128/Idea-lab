META-PROMPT: GENERAL GUIDELINES FOR ALL SECTIONS
================================================================================

You are an expert front-end developer specializing in creating clean, modern,
and responsive web components. When generating any website section, you must
adhere to the following principles:

TECHNICAL REQUIREMENTS:
- Use React with functional components and hooks
- Implement Tailwind CSS for styling (utility-first approach)
- Ensure full mobile responsiveness using Tailwind's responsive breakpoints
- Use semantic HTML5 elements for accessibility
- Follow React best practices (proper key props, clean component structure)
- Include proper TypeScript types if TypeScript is specified, otherwise use PropTypes or JSDoc comments

DESIGN PRINCIPLES:
- Maintain consistent spacing using Tailwind's spacing scale (4, 8, 12, 16, 24, 32, etc.)
- Use a cohesive color palette (primary, secondary, accent colors)
- Ensure proper contrast ratios for text readability (WCAG AA compliance)
- Implement smooth transitions and hover effects where appropriate
- Use modern design patterns (glassmorphism, subtle shadows, rounded corners)
- Maintain visual hierarchy through typography scale and spacing

CODE QUALITY:
- Write clean, readable, and maintainable code
- Use descriptive variable and function names
- Include comments for complex logic
- Ensure components are reusable and accept props for customization
- Avoid inline styles; use Tailwind classes exclusively
- Implement proper error handling and validation where needed

RESPONSIVE DESIGN:
- Mobile-first approach (design for mobile, enhance for desktop)
- Breakpoints: sm (640px), md (768px), lg (1024px), xl (1280px), 2xl (1536px)
- Test layouts at: 320px (mobile), 768px (tablet), 1024px (desktop), 1920px (large desktop)
- Ensure touch-friendly interactive elements (minimum 44x44px tap targets)
- Use flexbox and grid for layouts, avoid fixed widths

ACCESSIBILITY:
- Include proper ARIA labels where needed
- Ensure keyboard navigation works
- Maintain focus states for interactive elements
- Use semantic HTML elements
- Provide alt text for images

PROMPT 1: NAVIGATION BAR COMPONENT
================================================================================

Generate a fully responsive navigation bar component for a modern website with
the following specifications:

COMPONENT STRUCTURE:
- Component name: NavigationBar or Navbar
- Accept props: brandName (string), menuItems (array of objects with label and href), optional className
- Use React hooks (useState) for mobile menu toggle functionality

VISUAL DESIGN:
- Fixed or sticky positioning at the top of the page
- Clean, minimal design with subtle background (white/light with shadow or dark theme option)
- Brand name/logo on the left side, navigation links on the right (desktop)
- Smooth transitions for hover effects and mobile menu animations
- Use a modern color scheme (e.g., dark text on light background or vice versa)

DESKTOP LAYOUT:
- Horizontal layout with brand on left, navigation items on right
- Navigation links displayed in a row with appropriate spacing (gap-6 or gap-8)
- Hover effects: subtle color change or underline animation
- Active link state should be visually distinct

MOBILE RESPONSIVENESS:
- Hamburger menu icon (3 horizontal lines) visible on mobile/tablet (< 768px)
- Full-screen or slide-down mobile menu when hamburger is clicked
- Smooth animation for menu open/close (slide down or fade in)
- Mobile menu should overlay content or push content down
- Close button (X icon) in mobile menu
- Menu items stacked vertically in mobile view with adequate spacing
- Touch-friendly tap targets (minimum 48px height per menu item)

INTERACTIVE FEATURES:
- Mobile menu toggle functionality using useState hook
- Close menu when clicking outside (optional, but recommended)
- Smooth scroll to sections if using anchor links
- Active link highlighting based on current route/section

CODE REQUIREMENTS:
- Use Tailwind CSS classes exclusively
- Implement proper accessibility (aria-label, aria-expanded for mobile menu)
- Use semantic HTML (nav element, ul/li for menu items)
- Include proper event handlers for mobile menu toggle
- Clean component structure with separation of concerns

EXAMPLE PROPS STRUCTURE:
{
  brandName: "YourBrand",
  menuItems: [
    { label: "Home", href: "#home" },
    { label: "Services", href: "#services" },
    { label: "About", href: "#about" },
    { label: "Contact", href: "#contact" }
  ]
}

OUTPUT FORMAT:

- All Tailwind classes properly applied

- Ready to use with minimal or no modifications

PROMPT 4: CONTACT / LEAD FORM COMPONENT
================================================================================

Generate a professional contact form or lead generation form component with
proper validation structure, clean layout, and responsive design with the
following specifications:

COMPONENT STRUCTURE:
- Component name: ContactForm or LeadForm
- Accept props: onSubmit (function), optional className, optional fields (array for customization)
- Use React hooks (useState) for form state management
- Implement basic client-side validation

FORM FIELDS (Standard):
- Name (text input, required)
- Email (email input, required, email validation)
- Phone (tel input, optional, phone format validation)
- Subject (text input, optional)
- Message (textarea, required, minimum length validation)
- Submit button

VISUAL DESIGN:
- Clean, organized form layout with proper spacing
- Labels above or beside inputs (accessible label association)
- Input fields with consistent styling:
  * Padding: px-4 py-3
  * Border: 1px solid gray-300, rounded-lg
  * Focus state: border color change, ring/outline
  * Error state: red border, error message below field
- Textarea with adequate height (min-h-32 or min-h-24)
- Submit button: prominent, primary color, adequate padding, rounded corners
- Form container with max-width constraint (max-w-2xl) and centered layout

LAYOUT OPTIONS:
- Option A: Single column (all fields stacked, labels above)
- Option B: Two columns for name/email on desktop, single column on mobile
- Ensure form is centered on the page with appropriate padding

VALIDATION REQUIREMENTS:
- Required field validation (show error if empty on submit or blur)
- Email format validation (regex pattern)
- Phone format validation (optional, basic pattern)
- Message minimum length (e.g., 10 characters)
- Real-time or on-submit validation (show error messages)
- Error messages displayed below each field in red text (text-sm, text-red-600)

FORM STATE MANAGEMENT:
- Use useState for each field value
- Use useState for error states (object with field names as keys)
- Handle form submission with preventDefault
- Call onSubmit prop with form data
- Optional: Show success message after submission
- Optional: Reset form after successful submission

RESPONSIVE DESIGN:
- Single column layout on mobile
- Two-column layout for name/email on desktop (if using Option B)
- Input fields full width on mobile, appropriate width on desktop
- Submit button full width on mobile, auto width on desktop
- Adequate padding around form container on all devices

ACCESSIBILITY:
- Proper label association (htmlFor and id attributes)
- Required fields marked with aria-required="true"
- Error messages associated with fields using aria-describedby
- Keyboard navigation support
- Focus states clearly visible
- Submit button accessible via keyboard

STYLING DETAILS:
- Input focus: border-blue-500 or brand primary color, ring-2 ring-blue-200
- Input error: border-red-500, ring-2 ring-red-200
- Error message: text-red-600, text-sm, mt-1
- Submit button: bg-blue-600 hover:bg-blue-700 (or brand colors), text-white, font-semibold
- Disabled submit button state (while submitting)

CODE REQUIREMENTS:
- Use Tailwind CSS classes exclusively
- Semantic HTML (form element, proper input types, labels)
- React hooks for state management
- Validation functions (can be inline or separate)
- Clean component structure
- Proper event handlers (onChange, onSubmit, onBlur for validation)

  name: string,

  phone: string,
  subject: string,
  message: string
}

USAGE INSTRUCTIONS FOR AI TOOLS
================================================================================

When using these prompts with AI coding tools:

1. COPY THE ENTIRE PROMPT for the section you want to generate
2. PASTE into ChatGPT, Claude, Gemini, or your preferred AI tool
3. The AI should generate complete, production-ready code
4. If output needs refinement, add specific requests like:
   - "Make the colors match a blue and white theme"
   - "Use a different layout option (Option B instead of Option A)"
   - "Add animation to the mobile menu"
5. Test the generated code and iterate if needed

REFINEMENT TIPS:
- If colors don't match your brand, specify exact colors: "Use #3B82F6 for primary color"
- If spacing feels off, request: "Increase spacing between cards to gap-8"
- If mobile menu doesn't work, ask: "Fix the mobile menu toggle functionality"
- If form validation is missing, specify: "Add email format validation with regex"

These prompts are designed to be self-contained and should work consistently
across different AI tools. The key is the level of detail and structure
provided, which guides the AI to generate high-quality output.
